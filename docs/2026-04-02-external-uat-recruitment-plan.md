# my-agent 外部测试：邀请函与筛选标准

## 1. 测试者画像（Persona）
我们正在寻找符合以下特征的 1-3 名测试者：
- **技能背景**：熟练使用命令行（CLI），具备基础开发经验（了解 Python 环境配置）。
- **项目熟悉度**：**零基础**。从未参与过 my-agent 项目开发，不熟悉其内部架构。
- **需求痛点**：对“Agent 运行过程排查”、“对比不同运行结果”有实际兴趣或潜在需求的人员。

## 2. 邀请话术（草案）

> **标题：【邀请测试】参与 my-agent 核心诊断链路小规模 UAT**
>
> 你好！
>
> 我们正在开发一款名为 `my-agent` 的 AI Agent 运行框架，目前正处于核心诊断功能（Run -> Archive -> Compare）的验证阶段。
>
> 我们希望邀请你作为“外部观察者”参加一项极简测试。你不需要阅读任何架构文档，只需要尝试按照一份简单的《快速上手指南》，看看能否在 15 分钟内独立跑通我们的工具并给出直观感受。
>
> **你将获得：**
> - 抢先体验一款聚焦“诊断与对比”的 Agent 框架。
> - 我们的诚挚感谢，以及你的反馈将直接影响产品后续的字段定义与交互设计。
>
> **测试任务：**
> 1. 环境搭建与运行一条任务。
> 2. 查看运行归档（Archive）。
> 3. 进行两次运行结果的差异对比（Compare）。
>
> **参与方式：**
> 如果你有兴趣，请回复我，我将发送测试包和极简指南。
>
> 期待你的参与！

## 3. 筛选确认问题（用于初步筛选）
1. 你是否经常在终端（Terminal/CMD）中操作 Python 项目？
2. 你是否曾因为 AI Agent 输出不稳定而感到排查困难？
3. 你是否愿意在完全不听取作者解释的情况下，尝试独立跑通一个新工具？
